 [ ] Go to https://app.supabase.com
 [ ] Click SQL Editor
 [ ] Click "New Query"
 [ ] Copy this SQL and paste:
 
     ALTER TABLE pages ENABLE ROW LEVEL SECURITY;
     ALTER TABLE site_settings ENABLE ROW LEVEL SECURITY;
     ALTER TABLE nav_items ENABLE ROW LEVEL SECURITY;
     ALTER TABLE social_links ENABLE ROW LEVEL SECURITY;
     ALTER TABLE contact_messages ENABLE ROW LEVEL SECURITY;
     
     CREATE POLICY "Public pages" ON pages FOR SELECT USING (is_published = true);
     CREATE POLICY "Public settings" ON site_settings FOR SELECT USING (true);
     CREATE POLICY "Public nav" ON nav_items FOR SELECT USING (visible = true);
     CREATE POLICY "Public social" ON social_links FOR SELECT USING (true);
     CREATE POLICY "Public contact" ON contact_messages FOR INSERT WITH CHECK (true);

COMMON ISSUES & FIXES
═══════════════════════════════════════════════════════════════════════════════

ISSUE: "Policy already exists" error in SQL
→ SOLUTION: That's normal! Just means it was already there. Click Run anyway.

ISSUE: Still seeing console "Error fetching" messages
→ SOLUTION: 
   1. Refresh page: Ctrl+Shift+R
   2. Restart server: npm run dev
   3. Make sure SQL was actually run in Supabase

ISSUE: Settings won't save
→ SOLUTION:
   1. Check .env.local has SUPABASE_SERVICE_ROLE_KEY ✅ (it does)
   2. Restart server: npm run dev
   3. Try again

ISSUE: "Table not found" error
→ SOLUTION: Run sql/init_schema.sql first in Supabase
